Introduction

PABLO is a C++ library for octree mesh generation and management. PABLO allows parallel execution of two-dimensional and three-dimensional cases. PABLO performs a rapid parallel load balancing over the processes. The mesh can be built by imposing a 2:1 balancing only for the desired elements.

Tutorials

Some tutorials on the use of PABLO library are described here below. All the reults here illustrated are obtained by the test files in the repository. Here no code is shown, but one can follow the description of the tutorials by reading comments and code in the test files.

Tutorial Test 0

The first file test0.cpp is a simple tutorial about the instantation of octree objects. In this example two object of class Para_Tree are declared. The first one by using the default constructor, that builds an octree with one octant with size 1 and coordinates of node 0 in the origin. The second octree is instantiated by custom constructor, passing as input parameter the coordinates of node 0 equal to (10,20,0) and the size of the octant equal to 250. Note that PABLO works in a logical domain, the transformation from the logical domain to the physical one has to be implemented by the user in a particular class (Class_Map); by default a simple translation/scaling is supplied with the repository. The two octrees are saved in logical coordinates and in physical ones. The figure below shows both the octrees in logical domain.

Note that the two octants are exactly overlapped, in effect they have the same coordinates and size in the logical domain. In the next figure the same octrees are displayed in the physical domain.

Here the two octants are highly different in side (1/250) and for the coordinates of their node 0.
